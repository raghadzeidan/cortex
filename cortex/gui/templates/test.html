This is the GUI part of the Advanced System Deisgn Project.
Project brief summary: The main thing is a cognition snapshot, which includes user information and sequence of snapshots that include cognitions, specifically each snapshots includes information about the client's feelings, pose, image of what they're seeing and depth image of what they're seeing. as above, a client transmits this huge binary file with his cognitions to a server, a server then separates data via the cognition (he separates feelings data, pose data etc... from each snapshots), and then publishes them to a a message queue, which then 4 different parsers process the data given and re-publishes the process information (processed information for example is the binary data of the image, being processed into a .png file using pillow in python) into the message queue, which then a saver module saves the data in the database. to reflect the database, there is a restful-api, a cli (using click), and gui interface.
